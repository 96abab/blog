1. 시맨틱 태그의 목적?



-HTML5 이전에는 구조를 구분하기 위하여 div태그에 id 또는 클래스 등으로 구분하며 구조를 설계했으나

인터넷이 발전하면서 웹문서의 양이 많아지고 각자 생성 양식이 달라 원하는 문서를 찾기 점점 힘들어지게 되었다

               

div 태그와 같이 block element이면서 사이트의 구조를 설계하고 태그에 의미를 부여 하여 사이트의 구조를 파악하

기 편하게 만들어주어 좀더 명시적이고 직관적으로 설계할 수 있다

       

       

- SEO 최적화에 유리 (SEO: Search Engine Optimization)



검색 엔진이 태그 목적에 알맞게 설계되어있는 구조의 사이트에서 더욱 빠르고 효율적으로 정보를 파악할 수 있고 검색결과가

검색 엔진이 태그의 목적에 부합하게 설계되어있는 구조의 사이트에서 더욱 빨리 효율적으로 정보를 파악할 수 있어

검색 결과의 노출에 유리할 수 있게 해준다.



- 웹 접근성에 효율적



일반적인 브라우저에서는 차이가 없지만 스크린리더 시각장애인을 위한 웹 서핑

프로그램 과 같은 환경에서는 웹 접근성과 사용성을 향상시켜준다.

==
표준 동등 연산자(standard equality operator)
-형변환 및 비교 과정을 함께 수행한다
  

  1. 타입이 같고 값도 같으면 / true
  
  2. 타입이 같고 값이 틀리면 / false

  3. undefined,null 은 같은 것으로 한다 (undefined == null) / true
  
  4. 문자열,수가 같이 있다면 수로 형변환 한다음 비교한다
  예) 1== '1'  /  true

  5. 논리값일 때 true 수치는 1 false 는 0 으로 형변환 한다음 비교한다

  6. 객체 타입과 원시 타입이 같이 있으면 객체를 toString(),valueOf()
  메소드로 형변환 한다음 비교한다



===
완전 일치 연산자 (strict equality operator) 
-형변환을 수행하지 않고 엄격한 비교 과정을 수행한다

  
  == 표준동등연산자에서 true 여도 === 완전일치연산자에서는 대부분
  false 로 평가된다

  예) 1 === '1'  /  false, 1 === 1 / true



&&
비교대상이 둘다 true라면 true
예) 1 && 1  /  true
    1 && 0  /  false
    0 && 1  /  false
    0 && 0  /  false
